# Vibe Coding for Kids 战略定义

版本：v0.2  
定位：第一章第一节对齐版

## 1. 当前战略句

`Vibe Coding for Kids` 已升级为一级战略。  
当前先证明的不是“世界有多大”，而是孩子能否通过和固定搭档小精灵的对话，完成一节真实的系统修补闭环。

## 2. 这轮只验证什么

- 范围只看：希腊神话 MVP / 第一章 / 第一节
- 起点只看：“普罗米修斯盗火之后，人间初火系统失稳”
- 节型只看：修补裂缝节
- 系统对象只看：初火祭坛稳焰系统

## 3. 对第一节的直接产品约束

1. 主交互关系固定为 `孩子 <-> 小精灵`
2. 小精灵负责观察提醒、追问、镜像、缩小范围、把孩子的话落成系统动作
3. 小精灵绝不替孩子下结论，不直接给答案
4. 神话角色负责现场重量、情绪压力和世界反馈，不承担主教学职责
5. 扰动制造者负责拨动局部变量，不按传统大反派写法承担主戏
6. 开场 2 分钟内必须让孩子看见“半成功异常”，并愿意继续判断“火没到，还是火到了但不稳”

## 4. “对话驱动”在第一节里具体意味着什么

- 孩子先描述异常，再表达第一次判断
- 孩子说出想先查哪一部分
- 孩子说出想改什么，再触发运行
- 反馈必须同时有世界层和系统层
- 结论只能由孩子完成，NPC 只能帮他把话说清楚

## 5. 这轮不该做什么

- 不扩总世界观
- 不补总反派线
- 不把神话角色写成老师
- 不把对话做成聊天 UI 包装
- 不为了“自由感”放开无限闲聊

## 6. 本轮实际交付

本轮只交四个产物：

1. 小精灵设定卡
2. 灰烬拨针者设定卡
3. 第一节开场钩子草案
4. 世界语言与禁用表达
