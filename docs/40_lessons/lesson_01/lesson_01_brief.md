# 第一节正式 Brief

版本：v0.2  
状态：本轮聚焦交付  
所属体系：希腊神话 MVP / 第一章 / 第一节

## 1. 本轮锁定前提

- `Vibe Coding for Kids` 已升级为一级战略
- 孩子主要通过和固定搭档小精灵对话推进任务
- 神话角色不承担主教学职责，但可以保留重量和戏剧感
- 第一章扰动制造者是“条件拨针者”，临时代号“灰烬拨针者”
- 第一节从“普罗米修斯盗火之后，人间初火系统失稳”开始
- 第一节类型是“修补裂缝节”
- 第一节系统对象是“初火祭坛稳焰系统”

## 2. 产物 1：小精灵设定卡

### 角色定位

孩子的固定搭档。  
它不是老师，也不是答案机，而是把孩子带进异常现场、陪孩子把判断说清楚的“侦察型编程搭子”。

### 气质关键词

- 机灵侦察型
- 可信赖
- 反应快
- 会追问
- 说话轻盈但不碎
- 对异常敏感
- 绝不替孩子下结论

### 第一节里的产品职责

1. 第一时间感知到“稳焰系统有异常”
2. 把孩子带到火坛现场，并把注意力钉在“火已经到了，但它立不住”这件事上
3. 追问孩子看到的现象，而不是直接解释原因
4. 帮孩子把模糊判断收束成可执行的问题
5. 在第一次失败后，帮助孩子缩小范围，但不替孩子改答案

### 它应该怎么说

- “先别急着定原因，你先看见了什么？”
- “你觉得这是没点着，还是点着了但没站稳？”
- “如果先只查一处，你想先查哪一处？”
- “刚才那次失败，最像是哪一个条件还没站稳？”
- “我可以陪你缩小范围，但判断得你来做。”

### 它不应该怎么说

- “答案就是供氧不够。”
- “你应该先查封护。”
- “我帮你选好了。”
- “其实很简单，你照我说的改。”
- “不是这个，重新来。”

### 它与神话角色的关系

- 它是主对话对象
- 神话角色是现场重量和反馈来源
- 它可以帮孩子听懂现场，但不能替神话角色发言，也不能抢走神话角色的戏剧感

### 本轮暂不拍板

- 正式名字
- 具体外观语言
- 固定口头禅

## 3. 产物 2：灰烬拨针者设定卡

### 角色定位

第一章的扰动制造者。  
它不是“毁灭世界的大反派”，而是专门拨动局部条件、让原本能成立的系统进入临界失稳状态的人。

### 当前锁定身份

- 类型：条件拨针者
- 临时代号：灰烬拨针者
- 第一节中的职责：把初火祭坛稳焰系统中的某个关键条件拨歪

### 角色气质

- 冷静
- 克制
- 不喧哗
- 更像在“校偏”世界，而不是在“砸坏”世界
- 擅长制造“看起来像成功，其实没有真正成立”的异常

### 第一节中的有效作用

它在第一节里最重要的，不是露脸，而是制造一种足够清楚的异常：

- 火已经点着
- 亮度也出现了
- 但稳不住

这会逼孩子先做一个关键区分：

`这是“没有发生”，还是“发生了但没站稳”？`

### 设计边界

- 不把世界整体打碎
- 不制造压倒性的灾难
- 不抢走孩子和小精灵的对话中心
- 不通过大段独白解释自己

### 它留下的痕迹应该是什么感觉

- 条件被拨到了临界边界
- 现场像少了半格对劲
- 看上去不是彻底错误，而是“总差一点”
- 让人容易误判成“火种本身有问题”

### 本轮暂不拍板

- 它在第一节是否正面露面
- 它拨动的是哪一个具体条件
- 它和后续章节的连续关系

## 4. 产物 3：第一节开场钩子草案

### 开场目标

孩子必须在 2 分钟内同时获得三件事：

1. 看见一个很具体的异常
2. 听到一个自己能判断的问题
3. 感到“我再往下看，就能帮上忙”

### 共通钩子骨架

无论第一镜头最终放在孩子现实侧，还是直接切进火坛现场，进入异常现场后的节奏都应保持一致：

`火已经到了。`

`火已经亮了。`

`但火立不住。`

### 两分钟内的建议节奏

`0:00 - 0:20`  
孩子先接收到小精灵的异常提醒：这不是“没点着”，而是“点着了但没站稳”。

`0:20 - 0:50`  
画面给到第一座火坛。火焰忽明忽暗，亮一下，退一下，像刚要站住又被什么拨松了。

`0:50 - 1:20`  
神话侧给出重量反馈。可以是普罗米修斯的短句，也可以是现场人群的紧张反应，但只负责说明“火已送到，人却还没真正拥有稳定的火”。

`1:20 - 1:50`  
小精灵把问题交给孩子：  
“你先帮我定一下，这是火没到，还是火到了却没站稳？”

`1:50 - 2:00`  
孩子获得继续往下查的明确入口：先做第一次判断，再决定先查哪一处。

### 示例台词草案

小精灵：  
“先别把它当成熄了。你看，它亮了，可它立不住。”  
“问题不是有没有火，问题是哪个条件没让它站稳。”  
“第一下判断，你来定。”

神话侧短句示例：  
“火已经来到人间，可第一座火坛还没有真正稳住。”

### 这一钩子为什么能拉住孩子

- 它不是抽象危机，而是孩子一眼能看懂的异常
- 它不是大世界说明，而是一个孩子能立刻回答的判断题
- 它让孩子觉得自己不是旁观者，而是能帮系统站稳的人

## 5. 产物 4：世界语言与禁用表达

### 世界语言

第一节里的语言要服务判断、排查和重跑，不服务堆神话感。

推荐保留的表达方向：

- 用“站稳 / 立住 / 没站稳 / 松了一格”替代空泛的“失控 / 崩坏”
- 用“先看见什么，再定原因”替代“我知道问题在哪”
- 用“你想先查哪一处”替代“你应该去查”
- 用“哪个条件没站稳”替代“是不是某个人做错了”
- 用“重跑一次看看”替代“再试试运气”
- 用“这次告诉了我们什么”替代“这次失败了”

### 小精灵语言规范

- 句子要短，像侦察反馈，不像讲课
- 先指出现象，再追问判断
- 允许轻快，但不能碎嘴
- 可信赖，但不哄、不替答

推荐句式：

- “你先看。”
- “你先定。”
- “要不要先缩小范围？”
- “刚才那次，最像哪一处没站稳？”
- “我可以陪你查，但结论得你来下。”

### 神话角色语言规范

- 有重量，但句数要少
- 负责让现场变得重要，不负责把方法教完
- 可以传达“火对人间很重要”，但不能接管排查流程

推荐方向：

- “火已至，人间却还未真正留住它。”
- “这不是空无一物，而是差最后的稳住。”

### 禁用表达

- “答案是……”
- “你应该……”
- “其实很简单……”
- “你选错了。”
- “我来告诉你正确做法。”
- “我们先去打败反派。”
- “世界马上要毁灭了。”
- “普罗米修斯把火带坏了。”
- “你只要照做就行。”
- “随便试试，总会对。”

### 禁用原因

- 会把孩子从判断者降回执行者
- 会把第一节从修系统写成打 boss
- 会让神话重量压过系统判断
- 会让对话失去“建模与调试”的产品价值

## 6. 当前不拍板分歧点

下面几条都存在两种以上合理写法，而且会影响产品方向，这轮不自行拍板：

1. 第一镜头放在哪里  
   - 写法 A：先在孩子现实侧收到扰动，再进入火坛现场  
   - 写法 B：直接冷开到火坛现场，再由小精灵把孩子拉进判断  
   - 影响：会改变第一分钟的代入方式和 onboarding 节奏

2. 灰烬拨针者在第一节是否露面  
   - 写法 A：只留痕迹，不正面出现  
   - 写法 B：短暂露面，建立章节压感  
   - 影响：会改变第一章的悬念结构和反派存在感

3. 小精灵是否本轮就拍板名字与外观  
   - 写法 A：继续保留功能称呼，先把对话关系跑通  
   - 写法 B：现在就锁名字和视觉语言，提前建立记忆点  
   - 影响：会改变 IP 节奏和前端资产方向

4. 第一节具体被拨动的是哪一个稳焰条件  
   - 写法 A：封护条件被拨歪  
   - 写法 B：供氧边界被拨歪  
   - 写法 C：引燃顺序被拨歪  
   - 影响：会直接改变第一节的系统模块、反馈形式和对话回合骨架
